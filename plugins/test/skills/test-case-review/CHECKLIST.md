# 用例评审检查项

本文件提供 `test-case-review` 在 `review` 阶段使用的 4 维度详细检查项。SKILL.md 保留维度概览表，逐项检查时读取本文件。

## 需求覆盖率（P0）

- 逐个 RP-N 检查：是否有对应用例覆盖正向流程
- 反向/异常覆盖：每个 RP 是否有异常路径用例
- 设计稿对照（如有）：页面状态覆盖（默认态、空态、加载态、错误态）
- 输出覆盖映射表：RP-N → 用例列表 → 覆盖状态

## 场景完整性（P0）

- **端到端闭环**：操作是否走到终态（创建→成功提示→列表更新→详情正确）
- **跨场景联动**：分享/跳转/回调完整链路
- **边界值**：数值极值、字符串空/最大/特殊字符、集合空/满
- **异常路径**：断网/无权限/数据冲突/并发
- **状态流转**：合法+非法转换

## 用例正确性（P1）

- 预期结果是否与需求/设计稿一致
- 步骤顺序/可达性/完整性
- 优先级标注：核心流程 P0、重要功能 P1、一般功能 P2、极端边界 P3

## 用例规范性（P2）

- 命名清晰描述测试目的（非"测试编辑"、"功能验证1"）
- 步骤每步只做一件事、具体可执行
- 预期明确可验证（非"正常"、"成功"）
- 前置条件完整

## 输出格式

评审结果输出到 `review_result.md`。
