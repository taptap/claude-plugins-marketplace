## 📋 快速开始

### 1. 确认配置

确保已完成以下配置（参考 [../README.md](../README.md)）：

- ✅ Slack MCP（访问 Slack 消息）
- ✅ 飞书项目 MCP（创建 Bug 单）

向 Claude 发送以下任一命令：

```markdown
# 方式1：分析最新反馈
请按照 skills/feedback/SKILL.md 的流程，分析 #taptap-feedback 频道的最新反馈
```

## 🔄 工作流程

```
阶段1: 获取并分析反馈
  ├── 从 Slack 获取消息和线程回复
  ├── 提取关键信息（用户ID、设备、版本等）
  ├── 识别问题类型
  ├── 智能查询知识库（按需）
  └── 生成分析报告

阶段2: 输出报告
  └── 等待用户确认下一步

阶段3: 执行操作（用户确认）
  ├── 选项A：创建飞书 Bug（输入"创建飞书 Bug"）
  └── 选项B：直接回复（输入"回复消息：[内容]"）
```

## 🎯 核心特性

### 1. 智能知识库查询

根据问题类型**自动匹配**相关知识库：

| 关键词 | 查询文件 | 用途 |
|--------|---------|------|
| 实名、认证 | `实名问题.md` | 实名认证案例 |
| SDK、登录 | `TapSDK问题.md` | SDK 问题案例 |
| 下载、安装 | `游戏下载安装.md` | 下载安装案例 |
| 动态、内容 | `内容发布.md` | UGC 内容案例 |

**优势**：
- ✅ 按需查询，不盲目读取所有文件
- ✅ 匹配度 < 70% 时不查询知识库
- ✅ 提高分析速度和准确性

### 2. 自动创建飞书 Bug

支持一键创建飞书项目 Bug 单：

- ✅ 自动填充 Bug 信息（优先级、严重程度、描述等）
- ✅ 根据业务模块分配报告人和经办人
- ✅ 在 Slack 原消息下自动回复 Bug 链接

### 3. 线程上下文分析

使用 `slack_read_thread` 获取完整上下文：

- ✅ 查看线程中的所有回复
- ✅ 发现问题是否已处理
- ✅ 避免重复创建 Bug

## 📖 使用示例

### 示例 1：分析单条反馈

**用户输入**：
```
分析 https://xindong.slack.com/archives/C01EB1KAE4B/p1769651933515359
```

**AI 执行**：
1. 从 URL 提取 `channel_id` 和 `message_ts`
2. 调用 `slack_read_thread` 获取消息及回复
3. 识别关键信息：用户ID、设备、版本
4. 判断问题类型：动态显示异常 → 不匹配知识库
5. 检查线程回复：发现已有处理记录
6. 输出分析报告

**输出示例**：
```markdown
## 📋 反馈分析报告

### 基础信息
- 用户ID: 195775
- 设备: iPhone18,2
- 版本: 5.15.1

### Bug 判断
🟢 已确认为 Bug - 已修复，等待发版

### 当前状态
✅ Bug已创建：https://project.feishu.cn/pojq34/issue/detail/6751352355
✅ 代码已修复：张益珲已在新版本修复
✅ 发版计划：王伟栋确认今天发版

---
💡 无需额外操作，问题已在处理中
```

### 示例 2：创建 Bug 单

**用户输入**：
```
创建飞书 Bug
```

**AI 执行**：
1. 调用飞书项目 MCP `create_workitem`
2. 填充 Bug 信息（基于阶段1的分析）
3. 返回 Bug ID 和链接
4. 在 Slack 原消息下回复

**Slack 回复**：
```markdown
✅ Bug 单已创建

问题已记录到飞书项目：
🔗 [iOS端动态显示异常 - 6751352355](https://project.feishu.cn/pojq34/issue/detail/6751352355)

- 业务线：社区
- 优先级：P1
- 严重程度：一般
- 报告人：金诗晨（测试）
- 经办人：王伟栋（研发）
- 排查结论：是客户端 bug，需要客户端看下

可以通过链接跟踪 Bug 修复状态。
```

## 🚨 注意事项

### 1. 知识库查询规则

- ✅ **只有**明确匹配到问题类型时，才查询知识库
- ✅ 如果问题已在线程中得到处理，**不需要**再查知识库
- ❌ **不要**盲目读取所有知识库文件

### 2. Bug 创建规则

- ✅ **必须**先输出分析报告，等待用户确认
- ✅ **只有**用户明确输入"创建飞书 Bug"时，才创建
- ❌ **不要**在未确认的情况下自动创建 Bug

### 3. 线程分析规则

- ✅ **必须**使用 `slack_read_thread` 读取线程回复
- ✅ 检查问题是否已在线程中得到处理
- ❌ **不要**只看主消息就下结论
