# 审查清单

## 1. 安全性（阻塞）

必须在合并前修复的关键安全问题。

- [ ] **无硬编码密钥** - token、API key、密码、私钥
- [ ] **无 PII 泄露** - 日志中的邮箱、手机号、用户 ID
- [ ] **无 SQL 注入** - 查询中的字符串拼接
- [ ] **无命令注入** - 未过滤的输入传入 shell/exec
- [ ] **无路径穿越** - 用户输入用于文件路径但未验证
- [ ] **无 XSS 向量** - HTML/模板中未转义的用户输入
- [ ] **无不安全反序列化** - 不可信数据用于反序列化调用

## 2. 逻辑正确性（阻塞）

会导致运行时问题的 Bug 和逻辑错误。

### 空值安全
- [ ] 可空值使用前是否已检查？
- [ ] 可选字段是否正确处理？

### 边界条件
- [ ] **数组/切片**：空、单元素、越界访问
- [ ] **数值**：零值、负数、溢出、下溢
- [ ] **字符串**：空串、纯空白、Unicode、超长

### 状态与副作用
- [ ] 状态变更是否有意为之？
- [ ] 是否存在竞态条件？
- [ ] 共享状态是否正确同步？

### 错误处理
- [ ] 错误/nil 返回值是否正确处理？
- [ ] 错误信息是否有足够信息量？
- [ ] 错误传播是否正确？

### 控制流
- [ ] 循环是否可能无限运行？（缺少 break/终止条件）
- [ ] 循环中是否有 off-by-one 错误？
- [ ] break/continue/return 使用是否正确？

## 3. 资源管理（泄漏为阻塞）

资源泄漏会导致内存问题和连接耗尽。

### 文件与连接处理
- [ ] 打开的文件/连接是否已关闭？
- [ ] `defer`/`finally`/`using` 使用是否正确？
- [ ] 错误路径中资源是否已关闭？

### 内存管理
- [ ] 循环内是否有大量内存分配？
- [ ] 是否有无限增长的缓存或 map？
- [ ] 是否有阻止 GC 的循环引用？

### 并发资源
- [ ] goroutine/线程是否可能泄漏？
- [ ] 是否有正确的取消/超时机制？
- [ ] channel 是否适当关闭？

## 4. API 与兼容性（警告）

影响调用方的破坏性变更。

- [ ] 公共接口是否有**破坏性变更**？
- [ ] 是否保持了**向后兼容**？
- [ ] 是否引入或使用了**已废弃 API**？
- [ ] 与依赖的**版本兼容性**是否正确？

## 5. 代码质量（建议）

风格和可维护性问题。

- [ ] 无遗留的调试输出（`console.log`、`fmt.Println`、`print`）
- [ ] 无死代码/注释掉的代码
- [ ] 无魔法数字（应使用命名常量）
- [ ] 命名和格式一致
- [ ] 变量/函数名清晰
- [ ] 复杂逻辑有适当注释
