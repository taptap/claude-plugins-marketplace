# Feedback 模板集

本文件包含 feedback skill 使用的所有模板：Bug 标题/描述/评论、Slack 回复、分析报告格式。

---

## Bug 标题规范

**格式**：`[前缀][功能模块] 问题描述`

**前缀选项**：`[线上]` / `[回归]` / `[日志]` / `[兼容性]` / `[偶现]` / `[AutoTest]`

**标题模板**：
```
[线上][{业务模块}] {问题简述}
```

**标题示例**：
- `[线上][商店] 筛选功能异常 - 变更筛选条件后游戏列表不准确`
- `[线上][开发者后台] 测试资格明细平台信息只展示安卓端`
- `[线上][SDK-分享] iPhone17 Pro Max 分享页面显示异常`

## Bug 描述模板

用于 Open API 更新描述字段，纯文本 + 换行：

```
问题描述
{问题简述}

复现步骤
1. {步骤1}
2. {步骤2}

复现频次：{必现/偶现}

环境信息
设备: {设备}
版本: {版本}
用户ID: {用户ID}

相关链接
工单: {工单链接}
Slack: https://xindong.slack.com/archives/{channel_id}/p{msg_id去掉小数点}
```

## Bug 评论模板

用于 Open API 添加评论，纯文本 + 换行：

```
## 技术分析

Bug判断
{为什么判定为Bug，如：点赞是社区核心互动功能，出现明确的错误提示影响用户体验}

错误信息: {错误提示}

可能原因:
1. {可能原因1}
2. {可能原因2}

建议验证:
- {验证步骤1}
- {验证步骤2}
```

## Slack 回复模板（Bug）

```
✅ Bug 单已创建

<{飞书链接}|{Bug标题}>

优先级：{优先级}
业务线：{业务线}
经办人：{经办人}
```

**示例**：
```
✅ Bug 单已创建

<https://project.feishu.cn/pojq34/issue/detail/6764546357|[线上][开发者后台] 测试资格明细平台信息只展示安卓端，iOS/PC资格不显示>

优先级：P3
业务线：商店
经办人：刘峰
排查结论：是前端问题，需要前端排查下/暂时不能确定问题，先指给前端
```

## Slack 回复模板（需求向）

```
这个反馈看起来是需求向的问题，需要产品确认是否需要优化

@{产品经理} 请帮忙看下这个反馈：
- 用户反馈：{问题简述}
- 当前行为：{现有功能表现}
- 用户期望：{用户希望的行为}

请确认：
1. 这是否是预期行为？
2. 是否需要优化处理？
```

**示例**：

用户反馈：用户反馈排行榜数据更新太慢，希望能实时更新

分析：这是需求向问题，排行榜功能正常工作，用户希望优化更新频率

Slack 回复：
```
这个反馈看起来是需求向的问题，需要产品确认是否需要优化

@郑晨斓 请帮忙看下这个反馈：
- 用户反馈：排行榜数据更新太慢
- 当前行为：排行榜按现有周期更新
- 用户期望：希望能实时更新

请确认：
1. 这是否是预期行为？
2. 是否需要优化处理？
```

## 分析报告格式

```markdown
## 📋 反馈分析报告

### 基础信息
- **用户ID**: [ID]
- **设备**: [设备型号]
- **版本**: [版本号]
- **工单**: [Leanticket链接]
- **相关订单**: [订单ID]
- **Slack消息**: [Slack URL]

### 关键词识别
- [关键词1]、[关键词2]、[关键词3]

### 业务模块判断
✅ **所属模块**: [商店/社区/SDK/运营等]
✅ **责任端**: [iOS/Android/前端/后端]
✅ **负责团队**: [具体团队]
✅ **对接人**:
  - 测试负责人：[姓名]
  - 研发负责人：[姓名]

### 📚 历史案例参考（如有匹配的知识库）
**相似案例**: [案例描述]
**历史处理**: [处理方式]
**对接人**: [姓名]
**参考链接**: [Slack历史讨论]

### Bug 判断
🔴/🟡/🟢 **是否为 Bug**: [是/否/待确认]
- **判断依据**: [说明]
- **参考案例**: [知识库案例编号]

### 复现建议
**快速复现路径**：
1. [具体步骤1]
2. [具体步骤2]
3. [预期现象]

**复现关键点**：
- [需要注意的条件1]
- [需要注意的条件2]

### 技术分析
- **可能原因**: [分析 + 参考历史案例]
- **建议验证**: [如何验证]
- **相关接口/日志**: [技术信息]

---
💡 **下一步操作**：
- 如需创建 Bug 单，请回复："创建飞书 Bug"
- 如需直接回复用户，请回复："回复消息：[内容]"
```
